          Command Variables Help
          
        
          
          Variables let you reuse command templates across devices.
          Use placeholders like {{hostname}} in command lines,
          then provide values from global_vars or CSV host_vars.
        
          

        
          1) Placeholder Format
          
        
          Use double braces in commands:
          
        
          configure terminal
hostname {{hostname}}
clock timezone JST {{tz_offset}}
          

        
          2) Global Vars (job-level JSON)
          
        
          Set in the top input field: Global Vars (JSON).
          
        
          {
  "hostname_prefix": "edge",
  "tz_offset": "9"
}
          
        
          Equivalent API payload:
          
        
          {
  "job_name": "nightly rollout",
  "creator": "local",
  "global_vars": {
    "hostname_prefix": "edge",
    "tz_offset": "9"
  }
}
          

        
          3) Host Vars (per-device CSV)
          
        
          Use CSV column host_vars as a JSON object string:
          
        
          host,port,device_type,username,password,name,verify_cmds,host_vars,prod
10.0.0.1,22,cisco_ios,admin,pass,edge-1,show run,"{""hostname"":""edge-1"",""tz_offset"":""9""}",true
10.0.0.2,22,cisco_ios,admin,pass,edge-2,show run,"{""hostname"":""edge-2""}",false
          

        
          4) Resolution Priority
          
        
          
          If the same key exists in both places, device-level CSV value wins:
          host_vars > global_vars.
        
          

        
          5) Missing Variable Behavior
          
        
          
          If any placeholder has no value, preflight fails with HTTP 400.
          Device commands are not executed.
        
          

        
          6) Common Mistakes and Fixes
          
        
          - Invalid JSON in Global Vars:
  wrong: {"timezone":"Asia/Tokyo",}
  fix:   {"timezone":"Asia/Tokyo"}

- Global Vars must be an object:
  wrong: ["x", "y"]
  fix:   {"x":"1","y":"2"}

- CSV host_vars quoting:
  wrong: {"hostname":"edge-1"}   (not CSV-escaped)
  fix:   "{""hostname"":""edge-1""}"
          

        
          7) End-to-End Mini Example
          
        
          Inputs:
          
        
          global_vars:
{"tz_offset":"9","ntp_server":"192.0.2.10"}

command:
clock timezone JST {{tz_offset}}
ntp server {{ntp_server}}
hostname {{hostname}}

host_vars for 10.0.0.1:
{"hostname":"edge-1","ntp_server":"192.0.2.20"}
          
        
          Resolved commands for 10.0.0.1:
          
        
          clock timezone JST 9
ntp server 192.0.2.20
hostname edge-1
          

        
          8) Production Host Flag
          
        
          
          Optional CSV column prod marks production hosts.
          true enables production warning UI in Create/Monitor/Detail pages when selected targets include that host.
        
          
        
          host,port,device_type,username,password,name,verify_cmds,host_vars,prod
10.0.0.10,22,cisco_ios,admin,pass,core-prod,show run,"{""hostname"":""core-prod""}",true
10.0.0.20,22,cisco_ios,admin,pass,edge-dev,show run,"{""hostname"":""edge-dev""}",false
